=== PAZATOR CODEBASE MAP ===
Last updated: May 2026

yy PROJECT STRUCTURE yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy

pazator/                     Root project directory
  index.html                 Marketing landing page. Hero, SARPARAST/LCTX products,
                             Zor AI, CTXOD, design system.
  roadmap.html               Dev roadmap: 4 phases, 12 tasks, 18 gaps vs Gotham.
  google0f064d72b8389aee.html  Google Search Console verification.

  app/                       Main application (SPA shell)
    index.html               (2687 lines) Main app. Tabs, sidebar, graph, dashboard,
                             intel center, modals (humanModal, otherModal, chatUpload,
                             dataUpload, aiImport, contextModal, classifyModal,
                             cleanModal, settingsModal, aboutOverlay, etc.).
                             Logo dropdown with sync status card.

    stylez.css               (12593 lines) Dark theme. CSS vars in :root
                             (--primary-bg, --card-bg, --border-color,
                             --text-primary, --text-secondary, --text-muted,
                             --accent-light, --success, --warning, --danger,
                             --info, --btn-primary-bg, --panel-shadow, etc.).
                             Key classes: .panel, .tab-bar, .modal-content,
                             .clean-modal / .clean-modal-backdrop / .clean-modal-*
                             .logo-dropdown-menu / .logo-dropdown-item / .logo-card
                             .sync-config-content, .resolver-content,
                             .gis-toggle, .agent-panel, .toast-container,
                             .classification-banner, .facet-bar, .search-container.

    scriptz.js               (14640 lines) Main orchestration. Tab system
                             (initTabs, switchTab, closeTab at L~8000). Global
                             state (pazatorData, tags, cases, aiChatHistory).
                             Data loading (loadData, saveData, startAutoSave).
                             Virtual table (renderVirtualTable, renderPage).
                             ChatParser (L969), ChatStorageManager (L1155),
                             ChatAnalysisService (L1352). Agent init.

    yyy Core Data yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
    pazator_db.js            (184 lines) Legacy IndexedDB v1 (PazatorDB).
    pazator_engine.js        (297 lines) Main IndexedDB engine (PazatorV2 v2).
                             7 stores: humans, others, tags, cases, chats, meta,
                             relationships. CRUD: getAll, get, put, putMany,
                             remove, clear, count, bulkRemove, query.
    pazator_store.js         (390 lines) In-memory store + event system.
                             Holds humans/others/tags/cases/chats/relationships.
    pazator_objects.js       (851 lines) Ontology system. 20 default types
                             (gender, maritalStatus, nationality, etc.).
                             Autocomplete fields (L522).
    pazator_snappy.js        (191 lines) Snapshot utility. Up to 20 localStorage
                             snapshots. load/save/takeShot/clearShot.

    yyy Relationships & Heuristics yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
    pazator_relationships.js (260 lines) Relationship graph. 14 types:
                             friend, family, associate, employer, etc.
    pazator_heuristics.js    (164 lines) Identity resolution. Levenshtein,
                             nameSimilarity, checkSharedAttributes, scan.
    pazator_resolver.js      (447 lines) Entity resolver. Cache/history in
                             localStorage. resolveEntity, renderResolver,
                             mergeCandidates.

    yyy UI & Views yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
    pazator_ui.js            (327 lines) UI helpers (PazatorUI). debounce,
                             throttle, showModal (L779), drag helpers,
                             formatDate, escapeHtml, showFloatingNotification.
    pazator_facets.js        (251 lines) Faceted search. FacetIndex class:
                             build, getCounts, filterByFacets, search.
                             12 facet types + tags. Saved searches in localStorage.
    pazator_graphview.js     (1180 lines) D3.js force-directed graph.
                             Canvas + SVG modes. buildGraphData (L165),
                             startSimulation (L831), handleNodeClick (L772).
    pazator_dashboard.js     (622 lines) 7 widget types: entity-count, risk-pie,
                             activity-feed, graph-snapshot, case-status,
                             threat-timeline, ai-insights. Draggable layout.
    pazator_timeline.js      (199 lines) Per-entity timeline. getTimelineEvents
                             (L12): relationships, cases, chats, tracker, analysis.

    yyy AI / Intelligence yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
    pazator_gemini.js        (117 lines) Google Gemini HTTP client. 9 models.
                             chat() at L36. API key in localStorage.
    pazator_ai.js            (217 lines) AI Queue. concurrency control, caching,
                             streamChat (L131).
    pazator_agents.js        (964 lines) Agent analysis engine. Dynamic registry,
                             orchestrator, IntelligenceAgent. Defaults:
                             threat/risk/intel/connection.
    pazator_context.js       (143 lines) CTXOD tool system. 16 tools:
                             search_people, get_person, get_stats, etc.

    yyy Feature Modules yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
    pazator_sync.js          (529 lines) Server sync. Config modal (custom
                             self-contained modal, not cleanModal). Login/
                             register/logout with JWT. Push/pull data across
                             devices. Server health check, status info panel.
                             Vertically stacked login form with CSS variables.
                             Sync status dot in logo dropdown
                             (user@server + last sync).

    pazator_gis.js           (542 lines) GIS/map integration. MapLibre GL,
                             heatmap, geocoder, tracker overlay.
                             collectGeoEntities, loadConfig, saveConfig.

    pazator_pipelines.js     (696 lines) Pipeline builder. 4 connectors
                             (CSV/JSON/SQL/REST), 5 transforms (filter/map/
                             sort/limit/dedup), scheduled ingestion.
                             Run history. Uses cleanModal pattern.
    pazator_alerts.js        (692 lines) Alert engine. Rule-based + threshold
                             triggers. WebSocket push. Webhook delivery.
                             Channel: notification/websocket/webhook/all.
                             Uses cleanModal pattern with card-based rule UI.
    pazator_api.js           (634 lines) REST/GraphQL API. CRUD on all
                             entities. API key mgmt, webhook registry.
                             API console UI, webhook manager modal.
    pazator_ontology_designer.js (701 lines) Ontology type designer UI.
                             Add/rename/delete types, import/export JSON.
    pazator_workflow.js      (442 lines) TASTUR v2 workflow engine. 7 triggers
                             (tab_switch, data_added, threat_detected, etc.),
                             6 actions (popup, notify, toast, assign_tag,
                             open_tab, run_pipeline). WHEN/IF/THEN rules.
                             Event hooks into switchTab + store.emit.
    pazator_reports.js       (510 lines) Reporting. 6 widget types, drag-drop
                             builder, PDF/CSV export, scheduled auto-exports.

    yyy Infrastructure yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
    pazator_worker.js        (206 lines) Web Worker. Handlers: parse_csv,
                             calculate_credits, find_connections,
                             deduplicate_chats, search.
    walkthrough.js           (344 lines) Interactive tutorial overlay.
                             Step-by-step UI highlighting.

yy DATA PIPELINE yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy

  AI pipeline:  pazator_gemini.js → pazator_ai.js → pazator_agents.js
                → pazator_context.js → scriptz.js (chat UI)
  Agent system: pazator_agents.js (orchestrator → registry → agents)
  Dashboard:    pazator_dashboard.js (standalone, persisted layout)
  Ontology UI:  pazator_ontology_designer.js (standalone)
  Sync:         pazator_sync.js (JWT auth, push/pull, server health)
  GIS:          pazator_gis.js (MapLibre GL, geocoder, heatmap)
  Worker:       pazator_worker.js (CSV, credits, connections, dedup)
