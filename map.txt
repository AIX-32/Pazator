Last updated: June 2026 (10 performance improvements applied)

## Recent changes (June 2026):
  - JS logic → IndexedDB: cursor-advance pagination, index-based queries, kv store
  - Memory → Worker: graph layout, heuristics, bulk transform, aggregate stats
  - Full re-renders → Targeted: entity_changed events, aggregate cache, virtual lists
  - localStorage → IndexedDB (dual-write): chatHistory, findings, cases, recentSearches
  - Eager scripts → Lazy: 12 app modules removed from index.html, loaded on first tab/menu click
  - Unthrottled → rAF-throttled: resize handler
  - AI calls → Cached: credit scoring hash-keyed cache via AIQueue
  - PHASE 1 GOTHAM (current): Unified object model, Object Explorer, Entity Types in Ontology Designer
    - store.js: unified _objectIndex, getObjectById(), getRelatedObjects(), searchObjects()
    - objects.js: entity type CRUD (createEntityType, getEntityTypes, etc.)
    - object-explorer.js: Gotham-style full-page Object Explorer overlay
    - ontology.js: Entity Types section alongside Value Types
    - persistence.js: showObjectDetail() delegates to Object Explorer

yy DATA PIPELINE yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy

  AI pipeline:  js/ai/gemini.js → js/ai/ai.js (AIQueue, caching) → js/ai/agents.js
                → js/ai/context.js → js/ai/zor-chat.js (chat UI)
                Credits: AIQueue.getCached() before Gemini call (cacheKey = hash of person data)
  Agent system: js/ai/agents.js (orchestrator → registry → agents)
  Worker:       js/core/worker.js — handlers:
                  graph_tick       force-directed layout (repulsion + attraction)
                  heuristics_scan  name similarity + dedup
                  bulk_transform   tag extraction, age computation
                  aggregate_stats  threat counts, credit risk distribution
                  CSV (legacy), credits (legacy), connections (legacy), dedup (legacy)

  Storage layers (bottom → top):
    IndexedDB (engine.js)            Primary store. 7 object stores + _kvStore.
                                     Queries via getPage() (cursor-advance fast path)
                                     or index-based (getAllByIndex, countByIndex, queryRange).
    localStorage (fallback)          Used when pazatorStore unavailable (startup).
                                     migrateToIDB() copies pazator_* keys at init.
    In-memory cache (store.js)       _aggregateCache for precomputed stats.
                                     _messageBus (pub/sub) for entity_changed events.
    In-memory cache (chat-system.js) _cache with CACHE_TTL, _archiveCache.
                                     Dual-writes to IndexedDB _kvStore.

  Data flow:
    User action → store mutation → invalidateCache() + entity_changed event
      → persistence.scheduleRender() (coalesced DOM update)
      → visualization (no full re-render; only affected entities)

  Tab lazy-loading:
    ensureLazyModule() in tabs.js (dynamic <script> injection)
    analysis  → dashboard.js     ontology → ontology.js
    pipelines → pipelines.js     alerts  → alerts.js
    api       → api.js           workflow → workflow.js
    reports   → reports.js       sync    → sync.js (menu trigger)
    walkthrough                   snappy  → snappy.js (menu trigger)
    gis/resolver — not referenced by anything, never loaded

  JS-bottleneck → IndexedDB-bound:
    - Cursor-based pagination replaces full table scans
    - VirtualList replaces manual DOM construction for chat history & clusters
    - Worker thread handles layout/heuristics/aggregates off main thread
    - AIQueue caching avoids redundant Gemini calls
    - Dual-write pattern keeps synchronous localStorage reads while
      persisting asynchronously to IndexedDB (no sync IO on hot path)
