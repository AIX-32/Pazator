What Karline Is

Karline is Pazator's action planning system — a timeline-based operational board where you define, track, and verify actions. It bridges the gap between intelligence analysis and real-world operations: once you know what's happening, Karline helps you decide what to do about it.

Access it from the menu bar or the dedicated Karline tab. Actions are displayed on a scrollable day-column timeline and in a sidebar list.

Action Types

Every action has a type that determines how it's treated:

  • Static — fixed time and place. A surveillance shift, a meeting, a deadline. Has a start date and optional end date.
  • Dynamic — context-aware. An action that depends on evolving conditions, like "monitor for new chatter" or "re-assess when new data arrives".

Status Lifecycle

Actions move through four statuses, each color-coded in the timeline:

  • draft — being planned, not yet active
  • active — in progress
  • completed — finished
  • cancelled — no longer relevant

Use the Advance button in the detail view to move an action to its next status.

Zor Verification

Actions can be handed off to Zor for AI verification. Click Verify with Zor in an action's detail modal — the action details are sent to the configured AI provider (Gemini or DeepSeek) which reviews feasibility, completeness, and relevance. A real AI response determines if verification passes. Verified actions get a green checkmark in the timeline and a green left border on their card.

Zor can also create, update, and delete Karline actions via chat commands:

  • list_karline_actions — list all actions
  • create_karline_action — create with title, type, dates, goal, description, status
  • update_karline_action — update by ID
  • delete_karline_action — delete by ID

Evidence Linking

Actions can reference entities in Pazator as evidence. When creating or editing an action, click Select from entities to open a picker with search. Selected entities appear as tags. This connects operational actions back to the intelligence data that justifies them.

PDF Reports

Any action can be exported as a PDF report via the detail modal or right-click context menu. The report includes title, type, dates, status, verification status, goal, description, and linked evidence entities.

Timeline Navigation

  • Prev / Next — navigate in 14-day increments
  • Today — snap back to current date range
  • Day columns — each column shows cards for that date
  • Today column — highlighted in yellow

Filters

Use the toolbar toggles and sidebar controls to narrow down:

  • Static / Dynamic checkboxes — show/hide by type
  • Status filter — all / draft / active / completed / cancelled
  • Search — filter actions by title in the sidebar

Interaction

  • Click an action card — opens the detail modal
  • Right-click an action card — context menu: View Details, Edit, Download PDF, Delete
  • Hover an action card — subtle lift effect

Future

Planned improvements include shared action boards for multi-user ops, calendar sync, automated triggers that create actions from TIDE findings, and richer Zor integration for action validation and status suggestions.
